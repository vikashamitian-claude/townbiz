# BizTown Coding Rules

## Stack
- Godot 4.x
- GDScript
- 3D stylized low-poly (approved 2026-07-03, see HUMAN_DECISIONS.md; was "2D Isometric")
- Android First

## Rules

1. Never change the technology stack.
2. Never add features outside the assigned task.
3. Never redesign existing systems without approval.
4. Keep code simple and readable.
5. Prefer incremental improvements.
6. No unnecessary abstractions.
7. No premature optimization.
8. Inspect existing files before editing.
9. Never hallucinate files, APIs or systems.
10. Maintain Android touch compatibility.

## Workflow

Inspect → Plan → Implement → Test → Report

## Every task report must include

- Files inspected
- Files changed
- What was implemented
- How to test
- Known limitations
